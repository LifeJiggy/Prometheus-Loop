# Glossary

## A

**Adaptive Planning** — Agent's ability to create and modify plans based on context and outcomes.

**Agentic AI** — AI system that can plan, act, observe, learn, and iterate autonomously.

**Anomaly Detection** — Identifying unusual patterns in metrics or behavior.

**API (Application Programming Interface)** — Set of rules for software components to communicate.

## B

**Backoff Strategy** — Method for increasing delay between retry attempts.

**Blast Radius** — How many systems, users, or data records an action could affect.

**Byzantine Fault Tolerance** — System's ability to function correctly even when some components fail or act maliciously.

## C

**Caching** storing frequently accessed data for faster retrieval.

**Circuit Breaker** — Pattern that stops calling a failing service to prevent cascade failures.

**Consolidation** — Merging related memories to reduce size and improve retrieval.

**Context Window** — Maximum amount of text a language model can process at once.

## D

**Dead Letter Queue** — Storage for messages that couldn't be processed.

**Distributed System** — System running on multiple networked computers.

## E

**Evolution** — Agent's ability to acquire new capabilities over time.

**Exponential Backoff** — Retry strategy with exponentially increasing delays.

## F

**Fan-out/Fan-in** — Pattern where work is split across multiple agents and results are merged.

**Fault Tolerance** — System's ability to continue operating despite failures.

## G

**Goal Check** — Verification that the agent has achieved its objective.

**Graceful Degradation** — Continuing to operate at reduced capability when components fail.

## H

**Health Check** — Periodic verification that system components are functioning.

**Hierarchical Planning** — Planning at multiple levels of abstraction.

## I

**Idempotency** — Property where performing an operation multiple times has the same effect as doing it once.

## J

**Jitter** — Random variation added to retry delays to prevent thundering herd.

## K

**Knowledge Graph** — Graph-structured representation of entities and relationships.

## L

**Load Balancing** — Distributing work across multiple agents or resources.

## M

**Meta-Cognition** — Agent's ability to think about its own thinking.

**Memory Consolidation** — Process of merging and summarizing stored memories.

## N

**Non-Idempotent** — Operation that produces different results when performed multiple times.

## O

**Observability** — Ability to understand internal state from external outputs.

## P

**Prompt Injection** — Attack where malicious input hijacks agent behavior.

**Prompt Engineering** — Designing effective prompts for AI systems.

## Q

**Queue** — Data structure for storing messages in order.

## R

**RAG (Retrieval-Augmented Generation)** — Technique that retrieves relevant documents before generating responses.

**Rate Limiting** — Restricting number of requests per time period.

**Retry** — Attempting an operation again after failure.

## S

**Sandboxing** — Executing code in isolated environment to limit damage.

**Self-Healing** — Agent's ability to diagnose and fix failures automatically.

**Self-Healing** — Automatic recovery from errors without human intervention.

## T

**Token** — Unit of text that language models process (approximately 4 characters).

**Tokenization** — Process of converting text into tokens.

## V

**Vector Store** — Database optimized for storing and retrieving vector embeddings.

## W

**Workflow** — Sequence of steps to complete a task.
